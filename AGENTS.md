# AGENTS.md — vikunja-cli

This is a Python CLI project for managing Vikunja tasks and projects.

## Architecture

```
vikunja_client/     # Auto-generated from OpenAPI spec — DO NOT edit manually
vikunja_cli/        # CLI application layer
  main.py           # Typer app, registers command groups
  client.py         # get_client() factory — reads config, returns AuthenticatedClient
  config.py         # Config R/W: ~/.config/vikunja_cli/config.env
  formatters.py     # Rich table rendering helpers
  commands/
    auth.py         # vk auth login / show
    tasks.py        # vk tasks list/get/add/update/done/delete
    projects.py     # vk projects list/get/add/update/delete
openapi.json        # Swagger 2.0 spec (source)
openapi3.json       # OpenAPI 3.0 spec (used for codegen)
openapi-python-client.yaml  # literal_enums: true (required — spec has duplicate enum values)
pyproject.toml      # uv-managed, hatchling build, entry point: vk = vikunja_cli.main:app
```

## Package Management

This project uses **uv** exclusively. Do not use pip, virtualenv, or poetry.

```bash
uv sync                          # install / sync deps
uv add <package>                 # add runtime dep
uv add --group dev <package>     # add dev dep
uv run vk <command>              # run without activating venv
```

## Running the CLI

```bash
uv run vk --help
uv run vk tasks list
uv run vk projects list
```

Or with the installed symlink: `vk <command>` (requires `~/.local/bin` in PATH).

## Credentials

Stored in `~/.config/vikunja_cli/config.env`:

```
VIKUNJA_BASE_URL=https://your.vikunja.instance
VIKUNJA_API_TOKEN=your_token
```

Environment variables override the file. Never print or log the token.

## API Client

`vikunja_client/` is generated by `openapi-python-client` from `openapi3.json`.

- All API calls use `sync()` (synchronous httpx)
- `client.py` sets `verify_ssl=False` — the instance may have TLS quirks
- Error pattern: API errors return objects with `.message` but without `.id`; check with `hasattr(result, "id")` or `_check_error()` helper in each command module

## Adding New Commands

1. Add a new module in `vikunja_cli/commands/`
2. Import the Typer sub-app and register it in `main.py`
3. Use `get_client()` from `vikunja_cli.client` for all API calls
4. Use helpers from `vikunja_cli.formatters` for Rich output
5. Import API functions from `vikunja_client.api.<group>.<module>`
6. Import models from `vikunja_client.models.<model_name>`
7. Use `UNSET` (from `vikunja_client.types`) for optional fields not being set

## Regenerating the Client

When the API spec changes:

```bash
curl -s https://your.vikunja.instance/api/v1/docs.json -o openapi.json
swagger2openapi openapi.json -o openapi3.json
uv run openapi-python-client generate \
  --path openapi3.json \
  --meta none \
  --output-path vikunja_client \
  --config openapi-python-client.yaml
```

`openapi-python-client` is in the `dev` dependency group. The `literal_enums: true` config flag is **required** — without it, generation fails on duplicate enum values in the spec.

## Key Vikunja API Patterns

- **Create** uses HTTP `PUT` (not POST) — e.g. `PUT /projects/{id}/tasks`
- **Update** uses HTTP `POST` — e.g. `POST /tasks/{id}`
- **Delete** returns a `ModelsMessage` (not 204), check `.code` not status code
- `percent_done=100.0` + `done=True` together marks a task complete
- Due dates must be ISO 8601 with time: `2026-04-01T00:00:00Z`
- Pagination: pass `per_page=N`; response headers `x-pagination-total-pages` / `x-pagination-result-count`
